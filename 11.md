## 从 Motif 到 Footprint：为什么还不够

在第 10 篇中，你已经知道 GM12878 特异性 peak 中富集了 EBF1、PAX5 等 B 细胞 TF 的 motif。但这里有两个关键的逻辑缺口：

1. **Motif 存在 ≠ TF 结合。** 基因组中存在大量的 motif 匹配位点，但绝大多数并没有被对应的 TF 结合。
2. **Motif 相似性问题。** 同一家族的 TF（如 GATA1 和 GATA2）共享几乎相同的 motif，仅凭 motif 富集无法区分。

**TF footprinting** 利用了 ATAC-seq 数据的一个独特特性来解决这些问题：当一个 TF 结合在 DNA 上时，它会物理性地保护其结合位点免受 Tn5 转座酶的切割。这在 ATAC-seq 信号中形成一个局部的**切割频率下降**——即 "footprint"（足迹）。

💡 **类比理解**
> 想象下雨后的沙滩（= 开放染色质）。雨滴均匀地打在沙子上（= Tn5 切割），但如果有人站在沙滩上（= TF 结合），他脚下的那块沙子就不会被淋湿。这个干燥的"脚印"就是 footprint——**它证明有人（TF）确实站在那里。**

<!-- 图 1 位置：TF footprint 原理示意图。上方展示开放染色质区域中 TF 结合在 motif 处，Tn5 无法切割 TF 占据的位点；下方展示对应的 ATAC-seq 切割频率曲线在 motif 处形成 V 形凹陷 -->

**图 1：TF footprint 的形成原理。** 在开放染色质中，Tn5 均匀切割可及的 DNA（上），但被 TF 结合保护的区域切割频率下降，形成信号凹陷（下）。这个凹陷就是 footprint 的来源。

## Tn5 插入偏好性：一个必须解决的技术问题

Tn5 转座酶并不是完全随机地切割 DNA——它对切割位点两侧几个碱基的序列有偏好性。如果不校正这个偏好性，某些位置的信号升高或降低可能纯粹是 Tn5 偏好导致的，而非 TF 结合。

⚠️ **踩坑预警**
> 不校正 Tn5 偏好性是 footprint 分析中最常见的错误之一。在未校正数据上做 footprint 分析，可能产生大量假阳性和假阴性。所有主流 footprint 工具（TOBIAS、HINT-ATAC）都内置了偏好性校正步骤，**请务必使用。**

## TOBIAS：一站式 TF Footprint 分析工具

**TOBIAS**（Transcription factor Occupancy prediction By Investigation of ATAC-seq Signal）是目前最推荐的 ATAC-seq footprint 分析工具。它提供了从偏好性校正到差异结合检测的完整流程。

### 安装 TOBIAS

```bash
# ============================================================
# 安装 TOBIAS
# ============================================================
conda install -c bioconda tobias
```

### 流程概览

TOBIAS 的分析分为四步：

```
ATACorrect → FootprintScores → BINDetect → PlotAggregate
  (偏好性校正)    (足迹评分)    (差异检测)    (可视化)
```

下面我们逐步进行。

---

## 第一步：ATACorrect —— Tn5 偏好性校正

ATACorrect 会学习当前数据集中 Tn5 的序列偏好性，然后从信号中去除这个偏好。

```bash
# ============================================================
# TOBIAS ATACorrect —— 校正 Tn5 插入偏好性（GM12878）
# ============================================================
TOBIAS ATACorrect \
    --bam filtered/GM12878_rep1_final.bam \
    --genome ref/hg38.fa \
    --peaks peaks/consensus/consensus_peaks.bed \
    --outdir tobias/atacorrect/ \
    --read-shift 0 0 \
    --cores 8

TOBIAS ATACorrect \
    --bam filtered/GM12878_rep2_final.bam \
    --genome ref/hg38.fa \
    --peaks peaks/consensus/consensus_peaks.bed \
    --outdir tobias/atacorrect/ \
    --read-shift 0 0 \
    --cores 8
```

```bash
# ============================================================
# TOBIAS ATACorrect —— 校正 Tn5 插入偏好性（K562）
# ============================================================
TOBIAS ATACorrect \
    --bam filtered/K562_rep1_final.bam \
    --genome ref/hg38.fa \
    --peaks peaks/consensus/consensus_peaks.bed \
    --outdir tobias/atacorrect/ \
    --read-shift 0 0 \
    --cores 8

TOBIAS ATACorrect \
    --bam filtered/K562_rep2_final.bam \
    --genome ref/hg38.fa \
    --peaks peaks/consensus/consensus_peaks.bed \
    --outdir tobias/atacorrect/ \
    --read-shift 0 0 \
    --cores 8
```

参数说明：

| 参数 | 含义 |
|------|------|
| `--bam` | 输入 BAM（已在第 5 篇完成 Tn5 偏移校正的 final BAM） |
| `--genome` | 参考基因组 FASTA |
| `--peaks` | 分析区域——使用 consensus peak 集合 |
| `--outdir` | 输出目录 |
| `--read-shift` | 设为 `0 0` 跳过内置 Tn5 偏移，因为 BAM 已在第 5 篇完成校正 |
| `--cores` | 并行线程数 |

ATACorrect 会输出三个 bigwig 文件：

```
📊 输出：
tobias/atacorrect/
├── GM12878_rep1_final_uncorrected.bw   # 原始信号
├── GM12878_rep1_final_bias.bw          # Tn5 偏好性信号
└── GM12878_rep1_final_corrected.bw     # 校正后信号 ← 后续使用这个
```

💡 **校正前 vs 校正后**
> 你可以在 IGV 中同时加载 `_uncorrected.bw` 和 `_corrected.bw`，对比一个已知的 TF 结合位点（比如 CTCF motif 处）。校正后的信号在 footprint 位置会有更清晰的 V 形凹陷。

---

## 第二步：FootprintScores —— 计算 Footprint 分数

FootprintScores 对每个基因组位置计算一个 footprint score（足迹分数）。这个分数综合考虑了局部信号深度和信号凹陷的深度，反映了该位置被 TF 结合的可能性。

### 合并重复再打分

为了提高信噪比，我们先合并重复的校正信号，再进行打分：

```bash
# ============================================================
# 合并重复的校正信号
# ============================================================
# 方法：先合并 BAM，再一起做 ATACorrect
# 或者使用 deepTools bigwigAverage 合并校正后的信号

# 合并重复样本的信号（deepTools bigwigAverage）
bigwigAverage \
    -b tobias/atacorrect/GM12878_rep1_final_corrected.bw \
       tobias/atacorrect/GM12878_rep2_final_corrected.bw \
    -o tobias/atacorrect/GM12878_merged_corrected.bw \
    -p 8

bigwigAverage \
    -b tobias/atacorrect/K562_rep1_final_corrected.bw \
       tobias/atacorrect/K562_rep2_final_corrected.bw \
    -o tobias/atacorrect/K562_merged_corrected.bw \
    -p 8
```

```bash
# ============================================================
# TOBIAS FootprintScores —— 计算 footprint 分数
# ============================================================
TOBIAS FootprintScores \
    --signal tobias/atacorrect/GM12878_merged_corrected.bw \
    --regions peaks/consensus/consensus_peaks.bed \
    --output tobias/scores/GM12878_merged_footprints.bw \
    --cores 8

TOBIAS FootprintScores \
    --signal tobias/atacorrect/K562_merged_corrected.bw \
    --regions peaks/consensus/consensus_peaks.bed \
    --output tobias/scores/K562_merged_footprints.bw \
    --cores 8
```

```
📊 输出：
tobias/scores/
├── GM12878_merged_footprints.bw    # GM12878 足迹分数
└── K562_merged_footprints.bw       # K562 足迹分数
```

这些 bigwig 文件可以在 IGV 中可视化——分数越高的位置越可能有 TF 结合。

## 第三步：BINDetect —— 差异 Footprint 检测

这是 TOBIAS 最核心的一步。BINDetect 会在两个条件之间比较每个 TF motif 位点的 footprint 分数，从而判断哪些 TF 在某一条件中有更强的结合信号。

```bash
# ============================================================
# TOBIAS BINDetect —— 差异 TF footprint 检测
# ============================================================
TOBIAS BINDetect \
    --motifs ref/JASPAR2024_CORE_vertebrates.pfm \
    --signals tobias/scores/GM12878_merged_footprints.bw \
              tobias/scores/K562_merged_footprints.bw \
    --genome ref/hg38.fa \
    --peaks peaks/consensus/consensus_peaks.bed \
    --outdir tobias/bindetect/ \
    --cond_names GM12878 K562 \
    --cores 8
```

参数说明：

| 参数 | 含义 |
|------|------|
| `--motifs` | TF motif 数据库（JASPAR PFM 格式） |
| `--signals` | 两个条件的 footprint 分数 bigwig |
| `--genome` | 参考基因组 |
| `--peaks` | 分析区域 |
| `--cond_names` | 条件名称（用于输出标注） |

| TAL1 | K562 极显著 | K562 负分 | ✅ 一致 |

> 💡 **chromVAR（第 12 篇）的验证**
>
> 虽然本教程中 HOMER 和 TOBIAS 均未能产出实际结果，但我们在第 12 篇的 chromVAR 分析中成功验证了上述预期：GATA2、GATA1::TAL1 在 K562 中 TF 活性更高，IRF8/IRF9、REL/RELA 在 GM12878 中活性更高。这为上述交叉验证提供了独立证据。

**两种方法（motif 富集 + footprint）互相验证时，可大大增强结论的可信度。**

但 footprint 分析提供了 motif 富集无法给出的额外信息：

1. **结合 vs 未结合的区分**：motif 存在不等于被结合，footprint 能区分
2. **同家族 TF 的区分**：GATA1 和 GATA2 的 motif 极其相似，但 TOBIAS 通过 footprint 深度和位点数量可以提供更细的区分
3. **结合位点的精确定位**：BINDetect 输出的 bound BED 文件给出了每个 TF 在每个条件中的精确结合位点坐标

## 实用技巧与注意事项

### 数据深度要求

💡 **Footprint 分析对数据深度有较高要求**
> 一般建议每个样本至少 **50M 有效 fragments**（去重后）才能获得可靠的 footprint 信号。如果你的数据量较少（<30M），footprint 的 V 形凹陷可能不够清晰，建议考虑使用下一篇介绍的 chromVAR 方法。

### 批量可视化

如果你想对一组感兴趣的 TF 批量生成 footprint 图：

```bash

# 批量可视化 Top TF footprints
# ============================================================

# GM12878 富集的转录因子
for TF in EBF1 PAX5 SPI1 IRF4; do

        --TFBS tobias/bindetect/${TF}/beds/${TF}_GM12878_bound.bed \
              tobias/bindetect/${TF}/beds/${TF}_K562_bound.bed \
        --signals tobias/footprints/GM12878_merged_footprints.bw \
                  tobias/footprints/K562_merged_footprints.bw \
        --output figures/${TF}_footprint_comparison.pdf \
        --share_y both --plot_boundaries
done

# K562 富集的转录因子
for TF in GATA1 GATA2 TAL1 KLF1; do
    TOBIAS PlotAggregate \
        --TFBS tobias/bindetect/${TF}/beds/${TF}_K562_bound.bed \
              tobias/bindetect/${TF}/beds/${TF}_GM12878_bound.bed \
        --signals tobias/footprints/K562_merged_footprints.bw \
                  tobias/footprints/GM12878_merged_footprints.bw \
        --output figures/${TF}_footprint_comparison.pdf \
        --share_y both --plot_boundaries
done

# 通用因子
for TF in CTCF; do
    TOBIAS PlotAggregate \

              tobias/bindetect/${TF}/beds/${TF}_K562_bound.bed \

        --share_y both --plot_boundaries
done
```

### Footprint 分析的局限性

尽管 footprint 分析很强大，它也有局限：

1. **residence time 效应**：驻留时间短的 TF（如某些 pioneer factor 的快速扫描模式）可能不会留下清晰的 footprint
2. **小 motif 问题**：结合位点非常短的 TF（如 GATA 家族只有 ~6 bp）footprint 信号可能较弱
3. **计算资源需求**：完整的 TOBIAS 流程对大基因组数据集可能需要数小时
4. **需要多条件比较**：BINDetect 需要至少两个条件才能进行差异分析

## 替代工具与安装建议

TOBIAS 是目前最全面的 ATAC-seq footprinting 工具，但它的安装有时并不简单。以下是一些常见问题和替代方案：

### TOBIAS 安装注意事项

```bash
# ============================================================
# TOBIAS 安装建议
# ============================================================

# 方法 1：conda 安装（推荐）
conda create -n tobias python=3.8
conda activate tobias
conda install -c bioconda tobias

# 方法 2：pip 安装
pip install tobias

# 注意：
# - TOBIAS 依赖 xgboost，pip 安装时会自动安装 CPU 版本
# - GPU 支持（CUDA）是可选的，不是必需的——CPU 版本完全可以运行分析
# - 如果遇到编译错误，确保系统有 gcc/g++ 和相关开发库
# - Python 3.7-3.9 兼容性最好
```

### 替代 Footprinting 工具

如果 TOBIAS 安装困难，以下工具可以完成类似的分析：

| 工具 | 特点 | 安装难度 |
|------|------|---------|
| **HINT-ATAC** | RGT 工具套件的一部分，专门针对 ATAC-seq footprinting 优化 | 中等 |
| **Wellington** | pyDNase 包的一部分，经典的 DNase footprinting 算法，也可用于 ATAC-seq | 较低 |
| **deepTools + 手动分析** | 使用 `computeMatrix` + `plotHeatmap` 在已知 motif 位点处可视化 Tn5 切割模式 | 低（deepTools 通常已安装） |
| **chromVAR（第 12 篇）** | 虽然不做 footprinting，但通过 motif 可及性变异推断 TF 活性，是轻量替代方案 | 低（纯 R 包） |

> 💡 **Footprinting 是高级分析，对数据深度和工具安装都有较高要求。** 如果你的主要目标是识别谱系特异的 TF，motif 富集分析（第 10 篇）+ chromVAR（第 12 篇）的组合已经能提供非常有价值的信息。Footprinting 提供的额外"结合 vs 未结合"区分能力，可以在条件允许时作为进阶分析补充。
