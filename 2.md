## 步骤 1：用 Mamba 创建分析环境

### 为什么用 Mamba 而不是 Conda？

`mamba` 是 `conda` 的高性能替代品，使用 C++ 编写的依赖解析器，**速度比 conda 快 5–10 倍**。如果你还没有安装 `mamba`，推荐通过 [Miniforge](https://github.com/conda-forge/miniforge) 安装。

### 创建环境并安装命令行工具

```bash
# ============================================================
# 创建独立的 conda 环境，安装 ATAC-seq 分析所需的全部命令行工具
# 使用独立环境避免与其他项目产生依赖冲突
# ============================================================
mamba create -n atacseq python=3.10 -y

conda activate atacseq

mamba install -c bioconda -c conda-forge \
    fastqc=0.12.1 \
    multiqc=1.21 \
    fastp=0.23.4 \
    bowtie2=2.5.3 \
    samtools=1.19 \
    picard=3.1.1 \
    bedtools=2.31.1 \
    deeptools=3.5.5 \
    macs2=2.2.9.1 \
    homer=4.11 \
    idr=2.0.4.2 \
    tobias=0.16.1 \
    genrich=0.6.1 \
    sra-tools=3.0.10 \
    -y
```

💡 **为什么要锁定版本号？**
> 生信工具更新频繁，不同版本可能产生不同结果。在上面的安装命令中，我们为每个工具指定了版本号。如果你在复现时遇到问题，首先检查版本是否一致。

### 各工具的用途速览

| 工具 | 用途 | 使用篇目 |
|------|------|----------|
| `fastqc` | 原始数据质量检查 | 第 3 篇 |
| `multiqc` | 聚合多个 QC 报告 | 第 3、6 篇 |
| `fastp` | 接头去除和质量修剪 | 第 3 篇 |
| `bowtie2` | 序列比对到参考基因组 | 第 4 篇 |
| `samtools` | BAM 文件操作（排序、索引、过滤） | 第 4–6 篇 |
| `picard` | 标记 PCR 重复 | 第 5 篇 |
| `bedtools` | 基因组区间操作 | 第 5–9 篇 |
| `deeptools` | 信号可视化（bigWig、heatmap） | 第 6、14 篇 |
| `macs2` | Peak calling | 第 7 篇 |
| `homer` | Motif 富集分析和 peak 注释 | 第 8、10 篇 |
| `idr` | 重复性评估 | 第 8 篇 |
| `tobias` | TF footprinting 分析 | 第 11 篇 |
| `genrich` | ATAC-seq 专用 peak calling（备用） | 第 7 篇 |
| `sra-tools` | 从 SRA 下载数据（备用） | 第 2 篇 |

> 💡 **关于 Picard 和去重复**
> Picard 是 Java 工具，需要 JVM 运行环境。如果你的系统没有安装 Java 或希望避免 Java 依赖，可以使用 `samtools markdup` 作为替代方案，功能相同且无需 Java。**本教程在第 5 篇中使用 `samtools markdup` 进行去重复。**

> ⚠️ **关于 TOBIAS 的安装**
> TOBIAS 依赖较多（包括 MOODS、scipy、pysam 等），在某些系统上可能安装失败，特别是缺少特定编译工具或系统库的环境。如果安装遇到问题，建议使用独立的 conda 环境单独安装 TOBIAS，或参考 [TOBIAS GitHub](https://github.com/loosolab/TOBIAS) 的安装说明。

## 步骤 2：安装 R 包

部分下游分析（差异可及性、TF 活性、可视化）需要在 R 中完成。

```r
# ============================================================
# 安装 ATAC-seq 下游分析所需的 R/Bioconductor 包
# 包括差异分析、motif 分析、TF 活性推断和可视化
# ============================================================

# --- Bioconductor 包管理器 ---
if (!requireNamespace("BiocManager", quietly = TRUE))
    install.packages("BiocManager")

# --- 核心分析包 ---
BiocManager::install(c(
    "DiffBind",          # 差异可及性分析（第 9 篇）
    "ChIPseeker",        # Peak 基因组注释（第 8 篇）
    "chromVAR",          # TF 活性推断（第 12 篇）
    "motifmatchr",       # Motif 匹配（第 12 篇）
    "DESeq2",            # 差异分析引擎（第 9 篇）
    "GenomicRanges",     # 基因组区间操作
    "rtracklayer",       # 读写 BED/BigWig 文件
    "BSgenome.Hsapiens.UCSC.hg38",  # hg38 参考基因组序列
    "TxDb.Hsapiens.UCSC.hg38.knownGene",  # hg38 基因注释
    "org.Hs.eg.db",      # 人类基因 ID 映射
    "JASPAR2024",        # JASPAR 2024 motif 数据库
    "JASPAR2020",        # JASPAR 2020（chromVAR 兼容版本）
    "clusterProfiler"    # 功能富集分析（第 14 篇）
))

# --- 可视化包 ---
BiocManager::install(c(
    "ComplexHeatmap",    # 热图（第 9、14 篇）
    "ggplot2",           # 通用绑图
    "pheatmap"           # 快速热图
))
```

> ⚠️ **关于 JASPAR 版本的说明**
> 上面同时安装了 JASPAR2024 和 JASPAR2020。虽然 JASPAR2024 包含最新的 motif 数据，但 `chromVAR` 的 API 目前与 JASPAR2024 存在兼容性问题。**在第 12 篇 chromVAR 分析中，我们将使用 JASPAR2020 作为 motif 数据源。** 安装 JASPAR2024 不会造成冲突，其他分析（如 HOMER）不受影响。

## 步骤 3：下载 hg38 参考基因组索引

Bowtie2 比对需要预建的基因组索引。**自己从 FASTA 构建索引需要 ~1 小时，直接下载预建索引只需要几分钟。**

```bash
# ============================================================
# 下载 hg38 参考基因组的 Bowtie2 预建索引
# 使用 NCBI 提供的预建索引，节省构建时间
# ============================================================
mkdir -p ~/atacseq_tutorial/reference/bowtie2_index
cd ~/atacseq_tutorial/reference/bowtie2_index

# 从 NCBI 下载 hg38 (GRCh38) Bowtie2 索引
wget https://genome-idx.s3.amazonaws.com/bt2/GRCh38_noalt_as.zip

# 解压
unzip GRCh38_noalt_as.zip

# 检查索引文件
ls -lh GRCh38_noalt_as/*.bt2
```

```
📊 输出：
-rw-r--r-- 1 user user 3.8G  GRCh38_noalt_as.1.bt2
-rw-r--r-- 1 user user 2.8G  GRCh38_noalt_as.2.bt2
-rw-r--r-- 1 user user  54K  GRCh38_noalt_as.3.bt2
-rw-r--r-- 1 user user 2.8G  GRCh38_noalt_as.4.bt2
-rw-r--r-- 1 user user 3.8G  GRCh38_noalt_as.rev.1.bt2
-rw-r--r-- 1 user user 2.8G  GRCh38_noalt_as.rev.2.bt2
```

💡 **为什么用 `noalt` 版本？**
> hg38 包含大量 ALT contig（替代序列），如果使用完整版基因组，reads 可能比对到这些 ALT 区域，增加歧义比对。使用 `noalt` 版本可以避免这个问题，这也是 ENCODE 流程推荐的做法。此处下载的索引对应 ENCODE 使用的参考基因组 `GRCh38_no_alt_analysis_set_GCA_000001405.15.fasta`（ENCODE 文件编号 ENCFF110MCL）。

## 步骤 4：下载 ENCODE Blacklist 和基因组注释

### Blacklist 区域

**ENCODE blacklist** 是基因组中信号异常高的区域——这些区域在几乎所有测序实验中都会产生伪信号（比如着丝粒、端粒附近的重复区域）。**在 peak calling 之前必须去除这些区域。**

```bash
# ============================================================
# 下载 ENCODE blacklist v2 和基因组注释文件
# Blacklist 区域会在 peak calling 前被过滤，避免假阳性
# ============================================================
mkdir -p ~/atacseq_tutorial/reference/annotation
cd ~/atacseq_tutorial/reference/annotation

# 下载 ENCODE blacklist v2 (Boyle Lab)
wget https://github.com/Boyle-Lab/Blacklist/raw/master/lists/hg38-blacklist.v2.bed.gz
gunzip hg38-blacklist.v2.bed.gz

# 查看 blacklist 区域数量和大小
wc -l hg38-blacklist.v2.bed
awk '{sum += $3-$2} END {printf "Blacklist total size: %.2f Mb\n", sum/1e6}' hg38-blacklist.v2.bed
```

```
📊 输出：
910 hg38-blacklist.v2.bed
Blacklist total size: 36.09 Mb
```

### 基因组大小文件

```bash
# ============================================================
# 下载 hg38 chromosome sizes，后续 bedtools 和 deeptools 需要
# ============================================================
wget https://hgdownload.soe.ucsc.edu/goldenPath/hg38/bigZips/hg38.chrom.sizes

# 只保留标准染色体（去除 _random, _alt, chrUn 等）
grep -E '^chr[0-9]+\t|^chrX\t|^chrY\t|^chrM\t' hg38.chrom.sizes > hg38.standard.chrom.sizes

wc -l hg38.standard.chrom.sizes
```

```
📊 输出：
25 hg38.standard.chrom.sizes
```

## 步骤 5：从 ENCODE Portal 下载 ATAC-seq 数据

### 在 ENCODE Portal 上查找数据

1. 打开 [ENCODE Portal](https://www.encodeproject.org/)
2. 点击 **Data** → **Search**
3. 搜索条件设置：
   - **Assay**: ATAC-seq
   - **Biosample**: GM12878（或 K562）
   - **Genome Assembly**: GRCh38
   - **File type**: fastq
   - **Status**: released

💡 **如何确认数据质量？**
> ENCODE Portal 每个实验页面都有 **audit** 标记（绿色/橙色/红色）。优先选择全绿色（no warnings）的数据集。同时检查实验页面中的 quality metrics：FRiP > 0.3、TSS enrichment > 5 表示质量良好。

### 下载 FASTQ 文件

以下提供两种下载方式。推荐使用 `wget` 直接从 ENCODE 下载。

```bash
# ============================================================
# 从 ENCODE Portal 下载 GM12878 和 K562 ATAC-seq FASTQ 文件
# 每个样本包含 R1 和 R2 两个文件（双端测序）
# ============================================================
mkdir -p ~/atacseq_tutorial/raw_data
cd ~/atacseq_tutorial/raw_data

# ---- GM12878 rep1 (ENCSR637XSC, biorep1) ----
wget -c https://www.encodeproject.org/files/ENCFF063GWP/@@download/ENCFF063GWP.fastq.gz \
    -O GM12878_rep1_R1.fastq.gz
wget -c https://www.encodeproject.org/files/ENCFF095PJX/@@download/ENCFF095PJX.fastq.gz \
    -O GM12878_rep1_R2.fastq.gz

# ---- GM12878 rep2 (ENCSR637XSC, biorep2) ----
wget -c https://www.encodeproject.org/files/ENCFF587MZS/@@download/ENCFF587MZS.fastq.gz \
    -O GM12878_rep2_R1.fastq.gz
wget -c https://www.encodeproject.org/files/ENCFF930HSV/@@download/ENCFF930HSV.fastq.gz \
    -O GM12878_rep2_R2.fastq.gz

# ---- K562 rep1 (ENCSR483RKN, biorep1) ----
wget -c https://www.encodeproject.org/files/ENCFF391BFJ/@@download/ENCFF391BFJ.fastq.gz \
    -O K562_rep1_R1.fastq.gz
wget -c https://www.encodeproject.org/files/ENCFF186CQZ/@@download/ENCFF186CQZ.fastq.gz \
    -O K562_rep1_R2.fastq.gz

# ---- K562 rep2 (ENCSR483RKN, biorep2) ----
wget -c https://www.encodeproject.org/files/ENCFF440UAD/@@download/ENCFF440UAD.fastq.gz \
    -O K562_rep2_R1.fastq.gz
wget -c https://www.encodeproject.org/files/ENCFF350ZZR/@@download/ENCFF350ZZR.fastq.gz \
    -O K562_rep2_R2.fastq.gz
```

⚠️ **踩坑预警：ENCODE 文件编号可能变化**
> ENCODE Portal 可能在数据更新时更改文件编号（ENCFF ID）。如果上面的链接失效，请到 ENCODE Portal 按搜索步骤重新查找，获取最新的下载链接。`wget -c` 支持断点续传，网络中断后重新运行即可继续下载。

## 步骤 6：建立项目目录结构

一个清晰的目录结构是高效分析的基础。以下是本系列推荐的项目布局：

```bash
# ============================================================
# 创建标准化的项目目录结构
# 按分析步骤组织目录，输入和输出分开管理
# ============================================================
cd ~/atacseq_tutorial

mkdir -p raw_data          # 原始 FASTQ 文件
mkdir -p fastqc_raw        # 原始数据 FastQC 报告
mkdir -p clean_data        # 接头去除后的 FASTQ 文件
mkdir -p fastqc_clean      # 清洗后 FastQC 报告
mkdir -p alignment         # Bowtie2 比对结果 (BAM)
mkdir -p filtered_bam      # 过滤后的 BAM 文件
mkdir -p peaks             # Peak calling 结果
mkdir -p bigwig            # 归一化信号文件
mkdir -p motif             # Motif 分析结果
mkdir -p footprint         # Footprinting 结果
mkdir -p diff_analysis     # 差异可及性分析结果
mkdir -p chromvar          # chromVAR 分析结果
mkdir -p nucleosome        # 核小体分析结果
mkdir -p visualization     # 最终可视化图表
mkdir -p logs              # 运行日志
mkdir -p scripts           # 自定义脚本

# 查看目录结构
find . -maxdepth 1 -type d | sort | sed 's|^\./||' | head -20
```
