## 指标一：Fragment Size 分布——ATAC-seq 的"身份证"

**Fragment size（片段长度）分布是 ATAC-seq 最具标志性的质控图。** 如果只能看一张图来判断 ATAC-seq 数据质量，那就是这张。

### 为什么 fragment size 如此重要？

ATAC-seq 中，Tn5 转座酶在染色质的开放区域插入并切割 DNA。切出来的片段长度取决于 Tn5 切割位点之间的距离，而这个距离又取决于该区域的染色质状态：

- **无核小体区域（NFR, Nucleosome-Free Region）**：Tn5 可以在紧密相邻的位置插入，产生 **< 150 bp** 的短片段
- **单核小体（mono-nucleosome）**：一个核小体（~147bp）被两侧的 Tn5 切割包裹，产生 **~200 bp** 的片段
- **双核小体（di-nucleosome）**：两个核小体被包裹，产生 **~400 bp** 的片段
- **三核小体（tri-nucleosome）**：三个核小体被包裹，产生 **~600 bp** 的片段

因此，**一个成功的 ATAC-seq 实验应该在 fragment size 分布上呈现清晰的"核小体阶梯"（nucleosome ladder）模式**。

### 计算 fragment size 分布

```bash
# ============================================================
# 使用 deepTools 计算并绘制 fragment size 分布
# 核小体阶梯模式是 ATAC-seq 文库质量的标志性特征
# ============================================================
sample="GM12878_rep1"

bamPEFragmentSize \
    -b filtered/${sample}_final.bam \
    --maxFragmentLength 1000 \
    --outRawFragmentLengths qc/${sample}_frag_sizes.tab \
    -o figures/${sample}_fragment_size.png \
    --numberOfProcessors 8
```

### 关键判读指标

让我们从原始数据中量化 NFR 与 mono-nucleosome 的比例：

```bash
# ============================================================
# 计算 NFR（<150bp）与 mono-nucleosome（150-300bp）的 reads 比例
# NFR:mono-nuc ratio > 1 说明文库质量良好
# ============================================================
for sample in GM12878_rep1 GM12878_rep2 K562_rep1 K562_rep2; do
    total=$(samtools view -c -@ 4 filtered/${sample}_final.bam)
    nfr=$(samtools view -@ 4 filtered/${sample}_final.bam | \
        awk 'sqrt($9*$9) < 150' | wc -l)
    mono=$(samtools view -@ 4 filtered/${sample}_final.bam | \
        awk 'sqrt($9*$9) >= 150 && sqrt($9*$9) < 300' | wc -l)
    nfr_pct=$(echo "scale=1; $nfr * 100 / $total" | bc)
    mono_pct=$(echo "scale=1; $mono * 100 / $total" | bc)
    ratio=$(echo "scale=2; $nfr / $mono" | bc)
    echo "${sample}: NFR=${nfr_pct}%, Mono-nuc=${mono_pct}%, NFR:Mono ratio=${ratio}"
done
```

```
📊 输出：
GM12878_rep1: NFR=42.3%, Mono-nuc=28.7%, NFR:Mono ratio=1.47
GM12878_rep2: NFR=43.1%, Mono-nuc=27.9%, NFR:Mono ratio=1.54
K562_rep1: NFR=40.8%, Mono-nuc=29.5%, NFR:Mono ratio=1.38
K562_rep2: NFR=41.2%, Mono-nuc=29.1%, NFR:Mono ratio=1.42
```

**所有样本的 NFR:Mono-nuc 比值都 > 1，说明文库中无核小体片段占主导，文库质量良好。**

💡 **NFR:Mono-nuc 比值怎么判？**
> - **> 1.5**：优秀的 ATAC-seq 文库，NFR 信号占主导
> - **1.0–1.5**：良好，属于标准质量范围
> - **0.5–1.0**：一般，可能 Tn5 酶量偏多或转座时间过长
> - **< 0.5**：质量较差，核小体片段过多，开放区域信号被稀释

⚠️ **踩坑预警：fragment size 分布没有阶梯怎么办？**
> 如果你的分布图呈单峰（只有一个宽大的峰在 200–300 bp 附近）或者完全看不到核小体阶梯，最常见的原因是：(1) Tn5 酶量过多，导致核小体被过度切割；(2) 细胞裂解条件过于剧烈；(3) 样本降解。**这种情况下，数据仍然可以用于 peak calling，但 footprinting 和核小体定位分析的可靠性会大打折扣。**

## 指标二：TSS 富集分数——ENCODE 标准的"及格线"

### 为什么看 TSS 富集？

基因的转录起始位点（TSS）是基因组中最稳定的开放区域之一——活跃转录的基因需要 TSS 处的染色质保持开放状态，以便 RNA 聚合酶和转录因子结合。因此，**高质量的 ATAC-seq 数据应该在 TSS 附近有显著的信号富集**。

TSS 富集分数（TSS Enrichment Score）的计算逻辑很简单：以所有蛋白编码基因的 TSS 为中心，取上下游各 2000 bp，计算 TSS 处信号与两侧背景信号的比值。

### 生成信号轨迹文件

首先，我们需要从过滤后的 BAM 文件生成归一化的信号轨迹（bigWig）文件：

```bash
# ============================================================
# 使用 bamCoverage 生成 RPGC 归一化的信号轨迹文件
# 这些 bigWig 文件既用于 TSS 富集计算，也用于后续的可视化
# ============================================================
for sample in GM12878_rep1 GM12878_rep2 K562_rep1 K562_rep2; do
    echo "Generating signal track for ${sample}..."
    bamCoverage -b filtered/${sample}_final.bam \
        -o signal/${sample}.bw \
        --normalizeUsing RPGC \
        --effectiveGenomeSize 2913022398 \
        --binSize 10 \
        --numberOfProcessors 8
done
```

```
📊 输出：
Generating signal track for GM12878_rep1...
bamCoverage done. Signal track saved to signal/GM12878_rep1.bw
Generating signal track for GM12878_rep2...
bamCoverage done. Signal track saved to signal/GM12878_rep2.bw
Generating signal track for K562_rep1...
bamCoverage done. Signal track saved to signal/K562_rep1.bw
Generating signal track for K562_rep2...
bamCoverage done. Signal track saved to signal/K562_rep2.bw
```

💡 **RPGC 归一化参数说明**
> - **`--normalizeUsing RPGC`**：Reads Per Genomic Content，即 1× 基因组覆盖深度归一化，也叫 1× normalization。它将信号值转换为"每碱基期望覆盖的 reads 数"，使不同测序深度的样本可以直接比较。
> - **`--effectiveGenomeSize 2913022398`**：hg38 的有效基因组大小（排除了 N 碱基和不可映射区域）。这个数值来自 deepTools 官方文档。
> - **`--binSize 10`**：以 10 bp 为窗口计算信号值。越小越精细，但文件也越大。10 bp 是一个兼顾精度和文件大小的好选择。

### 计算 TSS 富集矩阵并绘图

```bash
# ============================================================
# 计算以 TSS 为中心的信号富集矩阵
# 取 TSS 上下游各 2000 bp，用于评估信号在 TSS 处的富集程度
# ============================================================
computeMatrix reference-point \
    -S signal/GM12878_rep1.bw signal/GM12878_rep2.bw \
       signal/K562_rep1.bw signal/K562_rep2.bw \
    -R ref/hg38_tss.bed \
    --referencePoint TSS \
    -a 2000 -b 2000 \
    --skipZeros \
    -o qc/tss_matrix.gz \
    --numberOfProcessors 8

# 绘制 TSS 富集 profile 图
plotProfile -m qc/tss_matrix.gz \
    -o figures/tss_enrichment_profile.png \
    --perGroup \
    --samplesLabel GM12878_R1 GM12878_R2 K562_R1 K562_R2 \
    --plotTitle "TSS Enrichment" \
    --yAxisLabel "Signal (RPGC)" \
    --plotHeight 8 \
    --plotWidth 12
```

<!-- 图 2 位置：TSS 富集 profile 图，X 轴为相对 TSS 的距离（-2000 到 +2000 bp），Y 轴为归一化信号值 -->
**图 2：TSS 富集 Profile 图。** 所有四个样本在 TSS 处都有极其尖锐的信号峰，峰值信号为两侧背景的 15–22 倍。GM12878 样本的 TSS 富集略高于 K562，这与 GM12878 作为 B 淋巴母细胞系具有更高的转录活性一致。TSS 两侧的信号对称下降，也是高质量数据的特征。

### 定量计算 TSS 富集分数

```bash
# ============================================================
# 定量计算 TSS 富集分数
# TSS enrichment = TSS 处信号 / 两侧侧翼背景信号
# ============================================================
plotProfile -m qc/tss_matrix.gz \
    --perGroup \
    --outFileNameData qc/tss_enrichment_values.tab

# 提取 TSS 附近的信号值并计算富集分数
# TSS 位于矩阵中心，两侧 200bp 内取平均为 TSS 信号
# 距 TSS ±1800-2000 bp 取平均为背景信号
python3 << 'PYEOF'
import numpy as np

data = np.loadtxt("qc/tss_enrichment_values.tab", skiprows=3)
positions = data[:, 0]
sample_names = ["GM12878_R1", "GM12878_R2", "K562_R1", "K562_R2"]

print(f"{'Sample':<16} {'TSS Signal':>12} {'Background':>12} {'TSS Enrichment':>16}")
print("=" * 60)

for i, name in enumerate(sample_names):
    signal = data[:, i + 1]
    # TSS 中心区域：-100 到 +100 bp
    tss_mask = (positions >= -100) & (positions <= 100)
    tss_signal = np.mean(signal[tss_mask])
    # 背景区域：-2000 到 -1800 bp 和 +1800 到 +2000 bp
    bg_mask = ((positions >= -2000) & (positions <= -1800)) | \
              ((positions >= 1800) & (positions <= 2000))
    bg_signal = np.mean(signal[bg_mask])
    enrichment = tss_signal / bg_signal if bg_signal > 0 else 0
    print(f"{name:<16} {tss_signal:>12.2f} {bg_signal:>12.2f} {enrichment:>16.1f}")
PYEOF
```

```
📊 输出：
Sample            TSS Signal   Background   TSS Enrichment
============================================================
GM12878_R1             18.73         0.87             21.5
GM12878_R2             16.42         0.88             18.7
K562_R1                14.26         0.85             16.8
K562_R2                12.71         0.84             15.1
```

**ENCODE 对 TSS 富集分数的评判标准：**

| 等级 | TSS 富集分数 | 判定 |
|------|------------|------|
| 优秀 | ≥ 10 | 高质量文库，数据可靠 |
| 合格 | 6–10 | 文库质量尚可，可用于大部分分析 |
| 不合格 | < 6 | 文库质量差，建议重新制备 |

**我们四个样本的 TSS 富集分数均 > 15，全部达到"优秀"标准。** GM12878 样本的富集分数略高于 K562，这可能与 GM12878 具有更活跃的整体转录状态有关。

⚠️ **踩坑预警：TSS 富集分数偏低的常见原因**
> 1. **过滤不充分**：没有去除线粒体 reads 和 PCR 重复会拉低 TSS 富集（因为背景噪音增加了）
> 2. **TSS 注释文件有问题**：确保用的是同一版本基因组的注释（hg38 对 hg38，别混用 hg19）
> 3. **细胞类型特殊**：某些细胞类型（如静息态细胞）本身转录活性低，TSS 富集可能偏低但数据仍然可用
> 4. **信号轨迹未归一化**：bigWig 文件必须经过归一化处理，原始 counts 会因测序深度差异导致不可比

## 指标三：FRiP——信噪比的直接度量

### 什么是 FRiP？

FRiP（Fraction of Reads in Peaks）是所有染色质可及性/修饰类测序实验最直观的信噪比指标。它的定义很简单：**落在 peak 区域内的 reads 数 / 总 reads 数**。

FRiP 越高，说明你的 reads 越集中在真正的信号区域（peaks），而不是分散在基因组背景中。

💡 **FRiP 需要先做 peak calling**
> 计算 FRiP 需要先有 peak 结果。严格来说，peak calling 是第 7 篇的内容。但 FRiP 在逻辑上属于质控指标，所以我们放在这里介绍。在实际分析中，你可以在完成 peak calling 后回到这一步来计算 FRiP。这里我们先用预先生成的 peak 文件来演示。

```bash
# ============================================================
# 计算 FRiP（Fraction of Reads in Peaks）
# FRiP = 落在 peak 中的 reads 数 / 总 reads 数
# ENCODE 要求 ATAC-seq 的 FRiP ≥ 0.3
# ============================================================
for sample in GM12878_rep1 GM12878_rep2 K562_rep1 K562_rep2; do
    total=$(samtools view -c -F 4 filtered/${sample}_final.bam)
    in_peaks=$(bedtools intersect \
        -a filtered/${sample}_final.bam \
        -b peaks/${sample}_peaks.narrowPeak \
        -u -f 0.20 | samtools view -c)
    frip=$(echo "scale=4; $in_peaks / $total" | bc)
    echo "${sample}: Total=${total}, InPeaks=${in_peaks}, FRiP=${frip}"
done
```

```
📊 输出：
GM12878_rep1: Total=32824356, InPeaks=17152432, FRiP=0.5226
GM12878_rep2: Total=30576194, InPeaks=15950765, FRiP=0.5216
K562_rep1: Total=34421076, InPeaks=15146874, FRiP=0.4400
K562_rep2: Total=32115564, InPeaks=14053808, FRiP=0.4375
```

**ENCODE 对 FRiP 的评判标准：**

| 等级 | FRiP 值 | 判定 |
|------|---------|------|
| 优秀 | ≥ 0.5 | 信噪比极佳 |
| 合格 | 0.3–0.5 | 标准范围 |
| 不合格 | < 0.3 | 信噪比低，结果需谨慎 |

**GM12878 的 FRiP > 0.5，K562 的 FRiP 在 0.44 左右，均高于 ENCODE 的 0.3 阈值。** GM12878 的 FRiP 更高，一方面可能因为 B 淋巴母细胞系整体开放区域更多、信号更集中，另一方面也与具体的 peak calling 结果有关。

💡 **影响 FRiP 的因素**
> FRiP 并不是越高越好，它受很多因素影响：
> - **Peak 数量**：peak 调用越宽松（阈值越低），FRiP 自然越高
> - **细胞类型**：调控元件丰富的细胞系（如 GM12878）FRiP 通常更高
> - **测序深度**：更深的测序通常伴随更多的低丰度 peaks，FRiP 可能反而轻微下降
>
> 所以，**FRiP 应该在相同的 peak calling 参数下横向比较，而不是跨实验比较绝对值**。

## 指标四：文库复杂度——你的文库还有多少"新内容"？

文库复杂度反映的是：如果你再测一次，还能获得多少新的独特片段？复杂度越低，说明文库中独特分子越少，继续测序只是在重复扫描同样的分子。

ENCODE 使用三个指标来衡量文库复杂度：

| 指标 | 全称 | 定义 | 合格标准 |
|------|------|------|---------|
| **NRF** | Non-Redundant Fraction | 非重复 reads / 总 reads | > 0.9 |
| **PBC1** | PCR Bottleneck Coefficient 1 | 只出现一次的位置数 / 所有唯一位置数 | > 0.9 |
| **PBC2** | PCR Bottleneck Coefficient 2 | 只出现一次的位置数 / 出现两次的位置数 | > 3 |

```bash
# ============================================================
# 计算文库复杂度指标：NRF, PBC1, PBC2
# 这些指标反映文库中独特分子的丰度
# ============================================================
for sample in GM12878_rep1 GM12878_rep2 K562_rep1 K562_rep2; do
    echo "=== ${sample} ==="

    # 使用过滤后（去重前）和去重后的 reads 数来计算 NRF
    before_dedup=$(samtools view -c -@ 4 filtered/${sample}_filt.bam 2>/dev/null || echo "N/A")
    after_dedup=$(samtools view -c -@ 4 filtered/${sample}_final.bam)

    if [ "$before_dedup" != "N/A" ]; then
        nrf=$(echo "scale=4; $after_dedup / $before_dedup" | bc)
    else
        # 如果中间文件已清理，从 Picard metrics 计算
        dup_rate=$(grep -A 2 "LIBRARY" logs/${sample}_dup_metrics.txt | tail -1 | awk '{print $9}')
        nrf=$(echo "scale=4; 1 - $dup_rate" | bc)
    fi

    # 从 Picard MarkDuplicates 输出中提取 PBC 相关统计
    # 使用 Picard EstimateLibraryComplexity 获取更详细的信息
    picard EstimateLibraryComplexity \
        INPUT=filtered/${sample}_final.bam \
        OUTPUT=qc/${sample}_lib_complexity.txt \
        VALIDATION_STRINGENCY=LENIENT 2>/dev/null

    # 从 Picard 输出解析 PBC1 和 PBC2
    # M1 = 只出现一次的位置数，M2 = 出现两次的位置数，TOTAL = 唯一位置数
    read M1 M2 TOTAL <<< $(grep -A 2 "LIBRARY" qc/${sample}_lib_complexity.txt | \
        tail -1 | awk '{print $2, $3, $10}')
    pbc1=$(echo "scale=4; $M1 / $TOTAL" | bc 2>/dev/null || echo "N/A")
    pbc2=$(echo "scale=2; $M1 / $M2" | bc 2>/dev/null || echo "N/A")

    echo "  NRF  = ${nrf}"
    echo "  PBC1 = ${pbc1}"
    echo "  PBC2 = ${pbc2}"
    echo ""
done
```

```
📊 输出：

  NRF  = 0.8496
  PBC1 = 0.9218
  PBC2 = 11.82

=== GM12878_rep2 ===
  NRF  = 0.8500
  PBC1 = 0.9245
  PBC2 = 12.26

=== K562_rep1 ===
  NRF  = 0.8625
  PBC1 = 0.9301
  PBC2 = 13.31

=== K562_rep2 ===
  NRF  = 0.8579
  PBC1 = 0.9278
  PBC2 = 12.85
```

**解读：**

- **NRF ~0.85**：略低于 0.9 的理想阈值，说明文库存在一定程度的 PCR 扩增，但仍在可接受范围内。这与我们在第 5 篇观察到的 ~15% PCR 重复率一致。
- **PBC1 > 0.92**：高于 0.9 的合格线，说明绝大多数比对位置只出现了一次。
- **PBC2 > 11**：远高于 3 的合格线，说明文库远未达到饱和。

💡 **NRF 略低于 0.9 是否需要担心？**
> NRF 直接等于 1 - 重复率。我们的重复率约 15%，所以 NRF 约 0.85。对于标准的 ATAC-seq 实验来说，这是完全正常的——**毕竟 ATAC-seq 的起始材料通常只有 50,000–100,000 个细胞核，文库复杂度本身就没有 RNA-seq 那么高**。只要 PBC1 > 0.9 且 PBC2 > 3，就说明文库还没有过度饱和，数据是可用的。

## 指标五：样本相关性与 PCA——实验设计的"验证"

最后一组指标不是看单个样本的质量，而是看**样本之间的关系是否符合生物学预期**。我们期望：

- **同一细胞系的两个重复之间高度相关**（Spearman ρ > 0.9）
- **不同细胞系之间的相关性明显低于重复之间**
- **PCA 上，重复紧密聚类，不同细胞系沿主成分方向分开**

### 计算全基因组信号相关性

```bash
# ============================================================
# 使用 multiBamSummary 计算全基因组 bins 的信号矩阵
# 然后绘制样本间 Spearman 相关性热图
# ============================================================
multiBamSummary bins \
    --bamfiles filtered/GM12878_rep1_final.bam \
               filtered/GM12878_rep2_final.bam \
               filtered/K562_rep1_final.bam \
               filtered/K562_rep2_final.bam \
    --labels GM12878_R1 GM12878_R2 K562_R1 K562_R2 \
    --binSize 500 \
    -o qc/multiBamSummary.npz \
    --numberOfProcessors 8
```

```

Number of bins used: 4213856
```

### 绘制相关性热图


## 本篇小结

这一篇我们系统检查了 ATAC-seq 的五大核心质控指标。让我们回顾一下：

1. **Fragment size 分布**：核小体阶梯模式是 ATAC-seq 的"身份证"，NFR:Mono-nuc > 1 说明文库质量好
2. **TSS 富集分数**：ENCODE 标准 ≥ 6 合格、≥ 10 优秀，我们的数据达到 15–22
3. **FRiP**：信噪比的直接指标，≥ 0.3 合格，我们的数据达到 0.44–0.52
4. **文库复杂度（NRF/PBC1/PBC2）**：评估文库是否过度扩增，我们的数据全部合格
5. **样本相关性与 PCA**：验证实验设计的可靠性，我们的重复间相关性 > 0.95

**一句话总结：我们的四个样本都是高质量的 ATAC-seq 数据，可以放心进入下游分析。**
