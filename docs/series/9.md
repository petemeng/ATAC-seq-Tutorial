## 为什么要做差异可及性分析？

在前两篇中，你分别对 GM12878 和 K562 做了 peak calling 和注释。你可能已经注意到，两种细胞的 peak 集合有很大的重叠——但也有不少差异。问题是：**仅仅因为一个 peak 在一个样本中存在、在另一个中不存在，就能说它"差异开放"吗？**

答案是**不能**。Peak calling 本身是一个二元决策（有/无），它不考虑信号强度的微妙差异，也不做统计检验来控制假阳性。**差异可及性分析的本质，是对每个基因组区域的 read count 进行统计建模和假设检验**——这和 RNA-seq 的差异表达分析在逻辑上是完全一样的。

| 概念 | RNA-seq DEG | ATAC-seq DA |
|------|:-----------:|:-----------:|
| 分析单元 | 基因 | Peak/基因组区间 |
| 计数方式 | exon reads → gene count | peak 内 reads → region count |
| 统计模型 | 负二项分布（NB） | 负二项分布（NB） |
| 标准化 | Library size + composition | Library size + composition |
| 多重检验校正 | BH/FDR | BH/FDR |

**关键差异**：ATAC-seq 的"特征"（peaks）是数据驱动的——它们本身就是从数据中发现的，这引入了一个选择偏差（ascertainment bias）。DiffBind 等工具会通过特定策略来缓解这个问题。

## 方法一：DiffBind 完整工作流程（推荐）

DiffBind 是 ATAC-seq/ChIP-seq 差异分析最常用的 R 包。它封装了从读取数据到统计检验的完整流程，内部调用 DESeq2 或 edgeR 作为统计引擎。

### Step 0: 准备 Sample Sheet

DiffBind 需要一个 CSV 格式的样本信息表。这是整个分析的入口。

```bash
# ============================================================
# 创建 DiffBind Sample Sheet
# ============================================================

cat > samplesheet.csv << 'EOF'
SampleID,Condition,Replicate,bamReads,Peaks,PeakCaller
GM12878_rep1,GM12878,1,filtered/GM12878_rep1_shifted.bam,peaks/macs2/GM12878_rep1_peaks.narrowPeak,narrow
GM12878_rep2,GM12878,2,filtered/GM12878_rep2_shifted.bam,peaks/macs2/GM12878_rep2_peaks.narrowPeak,narrow
K562_rep1,K562,1,filtered/K562_rep1_shifted.bam,peaks/macs2/K562_rep1_peaks.narrowPeak,narrow
K562_rep2,K562,2,filtered/K562_rep2_shifted.bam,peaks/macs2/K562_rep2_peaks.narrowPeak,narrow
EOF
```

Sample sheet 各列含义：

| 列名 | 含义 | 示例 |
|------|------|------|
| SampleID | 唯一样本标识 | GM12878_rep1 |
| Condition | 实验条件（分组） | GM12878 |
| Replicate | 重复编号 | 1 |
| bamReads | 过滤后的 BAM 文件路径 | filtered/GM12878_rep1_shifted.bam |
| Peaks | 对应的 peak 文件路径 | peaks/macs2/GM12878_rep1_peaks.narrowPeak |
| PeakCaller | peak 格式 | narrow |

```r
samples$contrasts
```

```r
# ---------- Step 5: 差异分析 ----------
samples <- dba.analyze(samples, method = DBA_DESEQ2)

# 查看结果概览
dba.show(samples, bContrasts = TRUE)
```

> 📊 **输出：**
> ```
>    Factor    Group Samples Group2 Samples2 DB.DESeq2
> 1 Condition GM12878       2  K562        2     28734
> ```

```r
# ---------- Step 6: 提取差异结果 ----------
res <- dba.report(samples,
                   method = DBA_DESEQ2,
                   th = 0.05,          # FDR 阈值
                   fold = log2(1.5))   # fold change 阈值 (|log2FC| > 0.585)

cat("\n===== 差异可及性分析结果 =====\n")
cat("总差异 peaks (FDR<0.05, |FC|>1.5):", length(res), "\n")

# 拆分方向
res_df <- as.data.frame(res)
gm_enriched <- sum(res_df$Fold > 0)
k562_enriched <- sum(res_df$Fold < 0)

cat("GM12878 富集 (更开放):", gm_enriched, "peaks\n")
cat("K562 富集 (更开放):", k562_enriched, "peaks\n")
```

> 📊 **输出：**
> ```
> ===== 差异可及性分析结果 =====
> 总差异 peaks (FDR<0.05, |FC|>1.5): 28734
> GM12878 富集 (更开放): 13256 peaks
> K562 富集 (更开放): 15478 peaks
> ```

**这些数字说明了什么？** 在 131,456 个 consensus peaks 中，约 22% 的区域在两种细胞间表现出显著的可及性差异。K562 富集的 peaks 略多于 GM12878，这与 K562 作为肿瘤细胞系整体染色质更开放的特征一致。

## 方法二：DESeq2 手动方法

如果你想更精细地控制分析流程，可以手动用 featureCounts 计数 + DESeq2 建模。这种方法更灵活，但步骤更多。

### Step 1: featureCounts 计数

```bash
# ============================================================
# featureCounts 在 consensus peaks 上计数
# ============================================================

# 把 consensus peaks 转为 SAF 格式
awk 'BEGIN{OFS="\t"; print "GeneID","Chr","Start","End","Strand"} 
     {print "peak_"NR, $1, $2, $3, "."}' \
    peaks/consensus/consensus_peaks.bed \
    > peaks/consensus/consensus_peaks.saf

# featureCounts 计数
featureCounts \
    -p --countReadPairs \
    -F SAF \
    -a peaks/consensus/consensus_peaks.saf \
    -T 8 \
    -o diffbind/featureCounts_results.txt \
    filtered/GM12878_rep1_shifted.bam \
    filtered/GM12878_rep2_shifted.bam \
    filtered/K562_rep1_shifted.bam \
    filtered/K562_rep2_shifted.bam
```

> 📊 **输出：**
> ```
> ||          Features : 131456
> ||       Annotations : consensus_peaks.saf
> ||     Paired-end : yes
> ||     Count ReadPairs : yes
> ||
> ||         Threads : 8
> ||           Level : meta-feature level
> ||      Multimapping : excluded
> ||
> || Process BAM file filtered/GM12878_rep1_shifted.bam...
> ||   Assigned fragments : 38924561 (74.3%)
> ||   Unassigned_NoFeatures : 12456823 (23.8%)
> ||   Unassigned_Ambiguity : 1003243 (1.9%)
> || ...
> ```

### Step 2: DESeq2 分析

```r
# ============================================================
# DESeq2 差异可及性分析——手动方法
# ============================================================

library(DESeq2)

# 读取 featureCounts 结果
fc <- read.table("diffbind/featureCounts_results.txt",
                  header = TRUE, row.names = 1, skip = 1)

# 提取计数矩阵（第 6 列开始是各样本的 counts）
counts <- fc[, 6:9]
colnames(counts) <- c("GM12878_rep1", "GM12878_rep2", "K562_rep1", "K562_rep2")

# 样本信息
coldata <- data.frame(
    condition = factor(c("GM12878", "GM12878", "K562", "K562")),
    replicate = factor(c(1, 2, 1, 2)),
    row.names = colnames(counts)
)

# 创建 DESeqDataSet
dds <- DESeqDataSetFromMatrix(
    countData = counts,
    colData = coldata,
    design = ~ condition
)

# 过滤低表达区域（至少 2 个样本中 count >= 10）
keep <- rowSums(counts(dds) >= 10) >= 2
dds <- dds[keep, ]
cat("Peaks after filtering:", nrow(dds), "\n")

# 运行 DESeq2
dds <- DESeq(dds)
res_deseq2 <- results(dds,
                        contrast = c("condition", "GM12878", "K562"),
                        alpha = 0.05)

summary(res_deseq2)
```

> 📊 **输出：**
> ```
> Peaks after filtering: 124873
>
> out of 124873 with nonzero total read count
> adjusted p-value < 0.05
> LFC > 0 (up, GM12878 enriched): 13812, 11%
> LFC < 0 (down, K562 enriched): 15934, 13%
> outliers [1]: 234, 0.19%
> low counts [2]: 18456, 15%
> (mean count < 12)
> ```

**注意**：手动 DESeq2 方法得到的差异 peaks 数量（~29,746）与 DiffBind 的结果（~28,734）接近但不完全相同。差异来自：（1）peak 集合的细微不同（DiffBind 会 re-center peaks）；（2）标准化策略的差异；（3）低表达过滤的阈值不同。**两种方法的结论是一致的，选择哪种取决于你对分析的控制粒度需求。**

## 生物学解读

数字本身不是目的——我们需要把差异 peaks 与已知的生物学联系起来。

### GM12878 富集 peaks 关联的生物学

GM12878 是一个 EBV 转化的 B 淋巴细胞系。我们期望在 GM12878 中更开放的区域与 **B 细胞发育和免疫功能** 相关。

```r
# ============================================================
# 生物学验证：差异 peaks 关联基因
# ============================================================

library(ChIPseeker)
library(TxDb.Hsapiens.UCSC.hg38.knownGene)
library(org.Hs.eg.db)
library(clusterProfiler)

txdb <- TxDb.Hsapiens.UCSC.hg38.knownGene

# 注释 GM12878 富集 peaks
gm_peaks <- res_df[res_df$Fold > 0, ]
gm_gr <- GRanges(seqnames = gm_peaks$seqnames,
                  ranges = IRanges(start = gm_peaks$start, end = gm_peaks$end))
export.bed(gm_gr, "diffbind/GM12878_enriched.bed")

gm_anno <- annotatePeak(gm_gr, TxDb = txdb, annoDb = "org.Hs.eg.db",
                          tssRegion = c(-3000, 3000))
gm_genes <- unique(as.data.frame(gm_anno)$SYMBOL)

# 注释 K562 富集 peaks
k562_peaks <- res_df[res_df$Fold < 0, ]
k562_gr <- GRanges(seqnames = k562_peaks$seqnames,
                    ranges = IRanges(start = k562_peaks$start, end = k562_peaks$end))
export.bed(k562_gr, "diffbind/K562_enriched.bed")

k562_anno <- annotatePeak(k562_gr, TxDb = txdb, annoDb = "org.Hs.eg.db",
                            tssRegion = c(-3000, 3000))
k562_genes <- unique(as.data.frame(k562_anno)$SYMBOL)

cat("GM12878 富集 peaks 关联基因:", length(gm_genes), "\n")
cat("K562 富集 peaks 关联基因:", length(k562_genes), "\n")

# 检查关键转录因子的基因座
key_tf <- data.frame(
    Gene = c("PAX5", "EBF1", "SPI1", "IRF4", "GATA1", "GATA2", "TAL1", "KLF1"),
    Expected = c("GM12878", "GM12878", "GM12878", "GM12878",
                 "K562", "K562", "K562", "K562")
)

for (i in 1:nrow(key_tf)) {
    gene <- key_tf$Gene[i]
    in_gm <- gene %in% gm_genes
    in_k562 <- gene %in% k562_genes
    status <- ifelse(in_gm & !in_k562, "GM12878 ✓",
              ifelse(!in_gm & in_k562, "K562 ✓",
              ifelse(in_gm & in_k562, "Both", "Neither")))
    cat(sprintf("  %s: %s (expected: %s)\n", gene, status, key_tf$Expected[i]))
}
```

> ```
> GM12878 富集 peaks 关联基因: 8234
> K562 富集 peaks 关联基因: 9156
>
>   PAX5: GM12878 ✓ (expected: GM12878)
>   EBF1: GM12878 ✓ (expected: GM12878)
>   SPI1: GM12878 ✓ (expected: GM12878)
>   IRF4: GM12878 ✓ (expected: GM12878)
>   GATA1: K562 ✓ (expected: K562)
>   GATA2: K562 ✓ (expected: K562)
>   TAL1: K562 ✓ (expected: K562)
>   KLF1: K562 ✓ (expected: K562)
> ```

**8/8 的关键转录因子基因座的可及性变化方向与预期完全一致！** 这是一个强有力的生物学验证：

- **GM12878 中更开放**：PAX5（B 细胞命运决定因子）、EBF1（早期 B 细胞因子）、PU.1/SPI1（髓系/B 细胞共用因子）、IRF4（浆细胞分化）
- **K562 中更开放**：GATA1（红系分化主调控因子）、GATA2（造血干细胞维持）、TAL1（红系/巨核系发育）、KLF1（β-globin 激活）

### GO 富集分析

## 实用提示与常见问题

### 1. 重复数不够怎么办？

差异分析的统计功效高度依赖于重复数。**2 个生物学重复是最低要求**（我们这次的数据就是 2 vs 2）。如果只有 1 个重复，DiffBind/DESeq2 无法估计组内方差，差异分析在统计上不可靠。

| 重复数 | 可用方法 | 建议 |
|:------:|---------|------|
| 1 vs 1 | 仅描述性比较，无法做统计检验 | 增加重复！ |
| 2 vs 2 | DiffBind/DESeq2（统计功效有限） | 可用但需谨慎解读 |
| 3 vs 3 | 标准分析（推荐最低标准） | 理想起步 |
| ≥4 vs 4 | 高功效分析 | 可以检测到微妙差异 |

### 2. FDR 和 Fold Change 阈值如何选择？

没有"标准答案"，但这里有一些指导：

- **探索性分析**（全面了解差异）：FDR < 0.1, |FC| > 1.2
- **标准分析**（本教程）：FDR < 0.05, |FC| > 1.5
- **严格分析**（高可信度子集）：FDR < 0.01, |FC| > 2.0

### 3. DiffBind vs DESeq2 手动方法的选择

| 特征 | DiffBind | DESeq2 手动 |
|------|:--------:|:-----------:|
| 易用性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 灵活性 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| Peak re-centering | ✅ 自动 | ❌ 需手动 |
| 多对比设计 | ✅ 方便 | ✅ 更灵活 |
| 可视化 | ✅ 内置丰富 | ❌ 需自己写 |
| 复杂实验设计 | 有限 | DESeq2 公式语法 |

**建议**：初学者用 DiffBind，想深入控制每个步骤的用 DESeq2 手动方法。两者得到的结论应该高度一致。
