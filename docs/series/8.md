## 为什么需要评估重复性？

在上一篇中，你拿到了每个样本的 peak 列表。但一个关键问题是：**两个生物学重复之间，哪些 peaks 是真实可靠的，哪些只是噪声？**

传统做法是简单地看两个重复的 overlap 比例——这很直观，但缺乏统计严谨性。**IDR 提供了一个更优雅的框架**：它不是简单地判定"有/无"，而是为每个 peak 赋予一个"不可重复概率"，从而定量地区分信号和噪声。

---

## 第一部分：IDR 分析

### IDR 的核心思想

IDR 的逻辑可以用一句话概括：**如果一个 peak 在两个重复中都排名靠前（信号强），那它更可能是真实的；如果它在一个重复中排名很高但在另一个中排名很低或根本不存在，那它更可能是噪声。**

IDR 模型假设数据来自两个组分的混合：
- **信号组分**：在两个重复中排名高度一致
- **噪声组分**：排名不一致，接近随机分布

### 运行 IDR

```bash
# ============================================================
# IDR 分析——评估生物学重复间的一致性
# ============================================================

mkdir -p idr

# 1. 先对 narrowPeak 文件按 p-value 排序（IDR 要求）
for sample in GM12878_rep1 GM12878_rep2 K562_rep1 K562_rep2; do
    sort -k8,8nr peaks/macs2/${sample}_peaks.narrowPeak \
        > peaks/macs2/${sample}_peaks_sorted.narrowPeak
done

# 2. GM12878 两个重复的 IDR
idr --samples \
    peaks/macs2/GM12878_rep1_peaks_sorted.narrowPeak \
    peaks/macs2/GM12878_rep2_peaks_sorted.narrowPeak \
    --input-file-type narrowPeak \
    --rank p.value \
    --output-file idr/GM12878_idr.txt \
    --plot idr/GM12878_idr_plot.png \
    --idr-threshold 0.05 \
    --log-output-file idr/GM12878_idr.log

echo "GM12878 IDR peaks (threshold 0.05): $(wc -l < idr/GM12878_idr.txt)"

# 3. K562 两个重复的 IDR
idr --samples \
    peaks/macs2/K562_rep1_peaks_sorted.narrowPeak \
    peaks/macs2/K562_rep2_peaks_sorted.narrowPeak \
    --input-file-type narrowPeak \
    --rank p.value \
    --output-file idr/K562_idr.txt \
    --plot idr/K562_idr_plot.png \
    --idr-threshold 0.05 \
    --log-output-file idr/K562_idr.log

echo "K562 IDR peaks (threshold 0.05): $(wc -l < idr/K562_idr.txt)"
```

> 📊 **输出：**
> ```
> Initial parameter values: [0.10 1.00 0.20 0.50]
> Final parameter values: [0.97 1.82 0.12 0.46]
> Number of peaks passing IDR cutoff of 0.05 - 72438/96238 (75.3%)
> GM12878 IDR peaks (threshold 0.05): 72438
>
> Initial parameter values: [0.10 1.00 0.20 0.50]
> Final parameter values: [0.96 1.75 0.14 0.48]
> Number of peaks passing IDR cutoff of 0.05 - 78254/108412 (72.2%)
> K562 IDR peaks (threshold 0.05): 78254
> ```

### 解读 IDR 结果

几个关键数字：

| 指标 | GM12878 | K562 |
|------|:-------:|:----:|
| Rep1 peaks | 96,238 | 108,412 |
| Rep2 peaks | 93,754 | 104,876 |
| 通过 IDR 0.05 | 72,438 (75.3%) | 78,254 (72.2%) |
| 信号比例（π₁） | 0.97 | 0.96 |

- **通过率 >60% 说明重复性良好**。我们的数据 72-75% 的 peaks 通过了 IDR，非常不错。
- **π₁（信号比例）接近 1** 表明绝大部分 peaks 来自信号组分——数据质量很高。
- 如果通过率 <50%，你可能需要回头检查数据质量或实验操作。

### IDR 图的解读

<!-- 图 1 位置：IDR 散点图，x 轴为 Rep1 peak rank，y 轴为 Rep2 peak rank，颜色表示 IDR 值 -->

**图 1：GM12878 IDR 散点图。** x 轴和 y 轴分别是两个重复中 peaks 的排名（-log10 p-value），颜色代表 IDR 值。左下角蓝色区域的 peaks 在两个重复中都排名靠前（IDR < 0.05），是可靠的信号；右上角红色区域的 peaks 排名不一致，是噪声。**理想的 IDR 图应该呈现清晰的对角线趋势**，大部分点集中在左下角。

## 第二部分：重复间 Overlap 分析

IDR 之外，直接看 overlap 比例也很有参考价值。

```bash
# ============================================================
# 重复间 Overlap 分析（bedtools intersect）
# ============================================================

mkdir -p overlap

# GM12878 重复间 overlap
bedtools intersect \
    -a peaks/macs2/GM12878_rep1_peaks.narrowPeak \
    -b peaks/macs2/GM12878_rep2_peaks.narrowPeak \
    -wa -u \
    | wc -l > overlap/GM12878_rep1_in_rep2.count

bedtools intersect \
    -a peaks/macs2/GM12878_rep2_peaks.narrowPeak \
    -b peaks/macs2/GM12878_rep1_peaks.narrowPeak \
    -wa -u \
    | wc -l > overlap/GM12878_rep2_in_rep1.count

echo "GM12878 Rep1 peaks in Rep2: $(cat overlap/GM12878_rep1_in_rep2.count) / 96238"
echo "GM12878 Rep2 peaks in Rep1: $(cat overlap/GM12878_rep2_in_rep1.count) / 93754"

# K562 同理
bedtools intersect \
    -a peaks/macs2/K562_rep1_peaks.narrowPeak \
    -b peaks/macs2/K562_rep2_peaks.narrowPeak \
    -wa -u \
    | wc -l > overlap/K562_rep1_in_rep2.count

bedtools intersect \
    -a peaks/macs2/K562_rep2_peaks.narrowPeak \
    -b peaks/macs2/K562_rep1_peaks.narrowPeak \
    -wa -u \
    | wc -l > overlap/K562_rep2_in_rep1.count

echo "K562 Rep1 peaks in Rep2: $(cat overlap/K562_rep1_in_rep2.count) / 108412"
echo "K562 Rep2 peaks in Rep1: $(cat overlap/K562_rep2_in_rep1.count) / 104876"
```

> 📊 **输出：**
> ```
> GM12878 Rep1 peaks in Rep2: 82456 / 96238 (85.7%)
> GM12878 Rep2 peaks in Rep1: 81293 / 93754 (86.7%)
> K562 Rep1 peaks in Rep2: 91748 / 108412 (84.6%)
> K562 Rep2 peaks in Rep1: 89536 / 104876 (85.4%)
> ```

**~85% 的 overlap 说明实验重复性非常好。** 注意 overlap 比例通常比 IDR 通过率高，因为它只要求空间位置有交集，而 IDR 还考虑了信号强度的一致性。

### UpSet Plot 可视化（R）

UpSet plot 是展示多个集合交集关系的利器，比 Venn 图更清晰。

```r
# ============================================================
# UpSet Plot — 可视化四个样本间的 peak 重叠
# ============================================================

library(GenomicRanges)
library(UpSetR)

# 读入四个样本的 peaks
read_peaks <- function(file) {
    df <- read.table(file, header = FALSE, sep = "\t")
    GRanges(seqnames = df$V1, ranges = IRanges(start = df$V2, end = df$V3))
}

peaks_list <- list(
    GM12878_R1 = read_peaks("peaks/macs2/GM12878_rep1_peaks.narrowPeak"),
    GM12878_R2 = read_peaks("peaks/macs2/GM12878_rep2_peaks.narrowPeak"),
    K562_R1    = read_peaks("peaks/macs2/K562_rep1_peaks.narrowPeak"),
    K562_R2    = read_peaks("peaks/macs2/K562_rep2_peaks.narrowPeak")
)

# 用 consensus 区间统计 overlap
consensus <- reduce(do.call(c, peaks_list))
overlap_mat <- sapply(peaks_list, function(pk) {
    as.integer(countOverlaps(consensus, pk) > 0)
})
rownames(overlap_mat) <- NULL

# UpSet plot
pdf("overlap/upset_peaks.pdf", width = 10, height = 6)
upset(as.data.frame(overlap_mat),
      sets = c("GM12878_R1", "GM12878_R2", "K562_R1", "K562_R2"),
      order.by = "freq", keep.order = TRUE,
      sets.bar.color = c("#4DAF4A", "#4DAF4A", "#E41A1C", "#E41A1C"),
      main.bar.color = "steelblue",
      text.scale = 1.3)
dev.off()
```

<!-- 图 2 位置：UpSet plot，展示四个样本 peaks 的交集关系。最大的柱子应该是"四个样本都有"的 peaks -->

**图 2：四个样本的 peak 重叠 UpSet plot。** 最高的柱子代表四个样本共有的 peaks（~65,000 个），这些是最可靠的开放区域。GM12878 特有或 K562 特有的 peaks 对应细胞类型特异性的染色质开放区域，也是后续差异分析的重点。

## 第三部分：ChIPseeker Peak 注释

找到了可靠的 peaks，下一个问题是：**这些开放区域落在基因组的什么地方？** 是在启动子？内含子？基因间区？这些信息对理解调控机制至关重要。

### 为什么注释很重要？

- **启动子区的 peaks** 通常对应活跃转录的基因
- **远端基因间区的 peaks** 往往是增强子（enhancer）
- **内含子中的 peaks** 可能是内含子增强子或剪接调控元件
- **注释分布本身就是数据质量的指标**：好的 ATAC-seq 数据应该有 ~30% 的 peaks 在启动子区

### 运行 ChIPseeker

```r
# ============================================================
# ChIPseeker Peak 注释
# ============================================================

library(ChIPseeker)
library(TxDb.Hsapiens.UCSC.hg38.knownGene)
library(org.Hs.eg.db)
library(clusterProfiler)
library(ggplot2)

txdb <- TxDb.Hsapiens.UCSC.hg38.knownGene

# 注释 consensus peaks
peakAnno <- annotatePeak(
    "peaks/consensus/consensus_peaks.bed",
    TxDb = txdb,
    annoDb = "org.Hs.eg.db",
    tssRegion = c(-3000, 3000),   # 定义 promoter 为 TSS ± 3kb
    level = "gene"
)

# 查看注释摘要
peakAnno
```

> 📊 **输出：**
> ```

> Genomic Annotation Summary:
>   Feature                               Frequency
>   Promoter (<=1kb)                       22.34%
>   Promoter (1-2kb)                        5.12%
>   Promoter (2-3kb)                        3.87%
>   5' UTR                                  1.23%
>   3' UTR                                  2.45%
>   1st Exon                                2.18%

>   1st Intron                             12.67%
>   Other Intron                           28.15%

**注释结果的关键解读：**

- **Promoter（启动子）占 ~31%**（22.3% + 5.1% + 3.9%）：这是 ATAC-seq 的典型特征，活跃的启动子区有大量 Tn5 切割
- **Intron（内含子）占 ~41%**（12.7% + 28.2%）：许多内含子增强子在这里被捕获
- **Distal intergenic（远端基因间区）占 ~18%**：这些主要是远端增强子
- **Exon（外显子）占 ~6%**：相对较少，因为外显子区域通常被核小体覆盖

## 实用提示

### 1. TSS Region 的定义

`tssRegion = c(-3000, 3000)` 定义了 promoter 的范围。你可以根据需要调整：

| 定义 | 范围 | promoter 占比 |
|------|------|:------------:|
| 窄 | ±1 kb | ~22% |
| 标准 | ±3 kb | ~31% |
| 宽 | ±5 kb | ~37% |

**建议使用 ±3 kb**，这是社区最常用的标准。

### 2. 注释优先级

当一个 peak 同时落在多个功能区域时（比如一个基因的内含子同时也是另一个基因的启动子），ChIPseeker 会按照以下优先级分配：

**Promoter > 5'UTR > 3'UTR > Exon > Intron > Downstream > Intergenic**

### 3. IDR 阈值选择

| 阈值 | 通过 peaks | 适用场景 |
|:----:|:---------:|---------|
| 0.01 | ~60,000 | 严格筛选，适合 motif 分析 |
| 0.05 | ~72,000 | 标准阈值（ENCODE 推荐） |
| 0.10 | ~80,000 | 宽松筛选，适合全面探索 |
