## 从 Motif 到 Footprint：为什么还不够

在第 10 篇中，你已经知道 GM12878 特异性 peak 中富集了 EBF1、PAX5 等 B 细胞 TF 的 motif。但这里有两个关键的逻辑缺口：

1. **Motif 存在 ≠ TF 结合。** 基因组中存在大量的 motif 匹配位点，但绝大多数并没有被对应的 TF 结合。
2. **Motif 相似性问题。** 同一家族的 TF（如 GATA1 和 GATA2）共享几乎相同的 motif，仅凭 motif 富集无法区分。

**TF footprinting** 利用了 ATAC-seq 数据的一个独特特性来解决这些问题：当一个 TF 结合在 DNA 上时，它会物理性地保护其结合位点免受 Tn5 转座酶的切割。这在 ATAC-seq 信号中形成一个局部的**切割频率下降**——即 "footprint"（足迹）。

💡 **类比理解**
> 想象下雨后的沙滩（= 开放染色质）。雨滴均匀地打在沙子上（= Tn5 切割），但如果有人站在沙滩上（= TF 结合），他脚下的那块沙子就不会被淋湿。这个干燥的"脚印"就是 footprint——**它证明有人（TF）确实站在那里。**

<!-- 图 1 位置：TF footprint 原理示意图。上方展示开放染色质区域中 TF 结合在 motif 处，Tn5 无法切割 TF 占据的位点；下方展示对应的 ATAC-seq 切割频率曲线在 motif 处形成 V 形凹陷 -->

**图 1：TF footprint 的形成原理。** 在开放染色质中，Tn5 均匀切割可及的 DNA（上），但被 TF 结合保护的区域切割频率下降，形成信号凹陷（下）。这个凹陷就是 footprint 的来源。

## Tn5 插入偏好性：一个必须解决的技术问题

Tn5 转座酶并不是完全随机地切割 DNA——它对切割位点两侧几个碱基的序列有偏好性。如果不校正这个偏好性，某些位置的信号升高或降低可能纯粹是 Tn5 偏好导致的，而非 TF 结合。

⚠️ **踩坑预警**
> 不校正 Tn5 偏好性是 footprint 分析中最常见的错误之一。在未校正数据上做 footprint 分析，可能产生大量假阳性和假阴性。所有主流 footprint 工具（TOBIAS、HINT-ATAC）都内置了偏好性校正步骤，**请务必使用。**

## TOBIAS：一站式 TF Footprint 分析工具

**TOBIAS**（Transcription factor Occupancy prediction By Investigation of ATAC-seq Signal）是目前最推荐的 ATAC-seq footprint 分析工具。它提供了从偏好性校正到差异结合检测的完整流程。

### 安装 TOBIAS

```bash
# ============================================================
# 安装 TOBIAS
# ============================================================
conda install -c bioconda tobias
```

### 流程概览

TOBIAS 的分析分为四步：

```
ATACorrect → ScoreBigwig → BINDetect → PlotAggregate
  (偏好性校正)    (足迹评分)    (差异检测)    (可视化)
```

下面我们逐步进行。

---

## 第一步：ATACorrect —— Tn5 偏好性校正

ATACorrect 会学习当前数据集中 Tn5 的序列偏好性，然后从信号中去除这个偏好。

```bash
# ============================================================
# TOBIAS ATACorrect —— 校正 Tn5 插入偏好性（GM12878）
# ============================================================
TOBIAS ATACorrect \
    --bam filtered/GM12878_rep1_shifted.bam \
    --genome ref/hg38.fa \
    --peaks results/consensus_peaks.bed \
    --outdir tobias/atacorrect/ \
    --cores 8

TOBIAS ATACorrect \
    --bam filtered/GM12878_rep2_shifted.bam \
    --genome ref/hg38.fa \
    --peaks results/consensus_peaks.bed \
    --outdir tobias/atacorrect/ \
    --cores 8
```

```bash
# ============================================================
# TOBIAS ATACorrect —— 校正 Tn5 插入偏好性（K562）
# ============================================================
TOBIAS ATACorrect \
    --bam filtered/K562_rep1_shifted.bam \
    --genome ref/hg38.fa \
    --peaks results/consensus_peaks.bed \
    --outdir tobias/atacorrect/ \
    --cores 8

TOBIAS ATACorrect \
    --bam filtered/K562_rep2_shifted.bam \
    --genome ref/hg38.fa \
    --peaks results/consensus_peaks.bed \
    --outdir tobias/atacorrect/ \
    --cores 8
```

参数说明：

| 参数 | 含义 |
|------|------|
| `--bam` | 输入 BAM（需为已移位校正的 shifted BAM，参考第 5 篇） |
| `--genome` | 参考基因组 FASTA |
| `--peaks` | 分析区域——使用 consensus peak 集合 |
| `--outdir` | 输出目录 |
| `--cores` | 并行线程数 |

ATACorrect 会输出三个 bigwig 文件：

```
📊 输出：
tobias/atacorrect/
├── GM12878_rep1_shifted_uncorrected.bw   # 原始信号
├── GM12878_rep1_shifted_bias.bw          # Tn5 偏好性信号
└── GM12878_rep1_shifted_corrected.bw     # 校正后信号 ← 后续使用这个
```

💡 **校正前 vs 校正后**
> 你可以在 IGV 中同时加载 `_uncorrected.bw` 和 `_corrected.bw`，对比一个已知的 TF 结合位点（比如 CTCF motif 处）。校正后的信号在 footprint 位置会有更清晰的 V 形凹陷。

---

## 第二步：ScoreBigwig —— 计算 Footprint 分数

ScoreBigwig 对每个基因组位置计算一个 footprint score（足迹分数）。这个分数综合考虑了局部信号深度和信号凹陷的深度，反映了该位置被 TF 结合的可能性。

### 合并重复再打分

为了提高信噪比，我们先合并重复的校正信号，再进行打分：

```bash
# ============================================================
# 合并重复的校正信号
# ============================================================
# 方法：先合并 BAM，再一起做 ATACorrect
# 或者使用 TOBIAS 内置的 merge 功能

# 简化处理：使用 bigwigMerge（deepTools）
bigwigMerge \
    tobias/atacorrect/GM12878_rep1_shifted_corrected.bw \
    tobias/atacorrect/GM12878_rep2_shifted_corrected.bw \
    tobias/atacorrect/GM12878_merged_corrected.bw

bigwigMerge \
    tobias/atacorrect/K562_rep1_shifted_corrected.bw \
    tobias/atacorrect/K562_rep2_shifted_corrected.bw \
    tobias/atacorrect/K562_merged_corrected.bw
```

```bash
# ============================================================
# TOBIAS ScoreBigwig —— 计算 footprint 分数
# ============================================================
TOBIAS ScoreBigwig \
    --signal tobias/atacorrect/GM12878_merged_corrected.bw \
    --regions results/consensus_peaks.bed \
    --output tobias/scores/GM12878_merged_footprints.bw \
    --cores 8

TOBIAS ScoreBigwig \
    --signal tobias/atacorrect/K562_merged_corrected.bw \
    --regions results/consensus_peaks.bed \
    --output tobias/scores/K562_merged_footprints.bw \
    --cores 8
```

```
📊 输出：
tobias/scores/
├── GM12878_merged_footprints.bw    # GM12878 足迹分数
└── K562_merged_footprints.bw       # K562 足迹分数
```

这些 bigwig 文件可以在 IGV 中可视化——分数越高的位置越可能有 TF 结合。

---

## 第三步：BINDetect —— 差异 Footprint 检测

这是 TOBIAS 最核心的一步。BINDetect 会在两个条件之间比较每个 TF motif 位点的 footprint 分数，从而判断哪些 TF 在某一条件中有更强的结合信号。

```bash
# ============================================================
# TOBIAS BINDetect —— 差异 TF footprint 检测
# ============================================================
TOBIAS BINDetect \
    --motifs ref/JASPAR2024_CORE_vertebrates.pfm \
    --signals tobias/scores/GM12878_merged_footprints.bw \
              tobias/scores/K562_merged_footprints.bw \
    --genome ref/hg38.fa \
    --peaks results/consensus_peaks.bed \
    --outdir tobias/bindetect/ \
    --cond_names GM12878 K562 \
    --cores 8
```

参数说明：

| 参数 | 含义 |
|------|------|
| `--motifs` | TF motif 数据库（JASPAR PFM 格式） |
| `--signals` | 两个条件的 footprint 分数 bigwig |
| `--genome` | 参考基因组 |
| `--peaks` | 分析区域 |
| `--cond_names` | 条件名称（用于输出标注） |

### BINDetect 输出结构

```
📊 输出：
tobias/bindetect/
├── bindetect_results.txt          # 所有 TF 的汇总统计表
├── bindetect_results.xlsx         # Excel 格式
├── GM12878_K562_overview.png      # 差异结合火山图
├── EBF1/
│   ├── beds/
│   │   ├── EBF1_GM12878_bound.bed    # GM12878 中 EBF1 结合位点
│   │   ├── EBF1_K562_bound.bed       # K562 中 EBF1 结合位点
│   │   └── EBF1_all.bed              # 所有 motif 匹配位点
│   └── plots/
│       └── EBF1_bindingscore_distribution.png
├── PAX5/
│   ├── beds/ ...
│   └── plots/ ...
├── GATA1/
│   ├── beds/ ...
│   └── plots/ ...
└── ... (每个 TF 一个子目录)
```

BINDetect 生成的火山图是 footprint 分析最重要的可视化。X 轴是差异结合分数（differential binding score），Y 轴是 -log10(p-value)。

> 3. **结合位点数**：Bound 列告诉你在每个条件中有多少个 motif 位点被判定为"有 TF 结合"

## 第四步：PlotAggregate —— 可视化特定 TF 的 Footprint

BINDetect 告诉你全局差异，PlotAggregate 让你对感兴趣的 TF 做精细可视化。

```bash
# ============================================================
# TOBIAS PlotAggregate —— 可视化 EBF1 footprint
# ============================================================
TOBIAS PlotAggregate \
    --TFBS tobias/bindetect/EBF1/beds/EBF1_GM12878_bound.bed \
    --signals tobias/atacorrect/GM12878_merged_corrected.bw \
              tobias/atacorrect/K562_merged_corrected.bw \
    --output figures/EBF1_footprint.pdf \
    --share_y both \
    --plot_boundaries
```

```bash
# ============================================================
# TOBIAS PlotAggregate —— 可视化 GATA1 footprint
# ============================================================
TOBIAS PlotAggregate \
    --TFBS tobias/bindetect/GATA1/beds/GATA1_K562_bound.bed \
    --signals tobias/atacorrect/GM12878_merged_corrected.bw \
              tobias/atacorrect/K562_merged_corrected.bw \
    --output figures/GATA1_footprint.pdf \
    --share_y both \
    --plot_boundaries
```

```bash
# ============================================================
# TOBIAS PlotAggregate —— 可视化 CTCF footprint（组成型对照）
# ============================================================
TOBIAS PlotAggregate \
    --TFBS tobias/bindetect/CTCF/beds/CTCF_GM12878_bound.bed \
    --signals tobias/atacorrect/GM12878_merged_corrected.bw \
              tobias/atacorrect/K562_merged_corrected.bw \
    --output figures/CTCF_footprint.pdf \
    --share_y both \
    --plot_boundaries
```

<!-- 图 3 位置：三个 TF 的 aggregated footprint 曲线图，左 EBF1（GM12878 深 V 形凹陷，K562 几乎平坦）、中 GATA1（K562 深凹陷，GM12878 平坦）、右 CTCF（两者凹陷深度相似） -->

**图 3：三个代表性 TF 的 aggregated footprint 曲线。** 左：EBF1 在 GM12878 中有深V形 footprint，K562 中信号平坦——说明 EBF1 在 GM12878 中结合而在 K562 中未结合。中：GATA1 正好相反。右：CTCF 在两个细胞系中 footprint 深度相近——说明 CTCF 为组成型结合，不受谱系影响。

⚠️ **踩坑预警**
> PlotAggregate 的 `--TFBS` 参数需要的是 BINDetect 输出中某一条件的 bound 位点 BED 文件，而不是所有 motif 匹配位点。如果使用 `_all.bed`，footprint 信号会被非结合位点稀释，V 形凹陷会变浅甚至消失。

## 与 Motif 富集结果的交叉验证

现在让我们把第 10 篇（HOMER motif 富集）和本篇（TOBIAS footprint）的结果放在一起：

| TF | HOMER 富集 (Ch10) | TOBIAS Footprint (Ch11) | 一致性 |
|----|-------------------|------------------------|--------|
| EBF1 | GM12878 p=1e-450 | GM12878 score=+0.42 | ✅ 一致 |
| PAX5 | GM12878 p=1e-320 | GM12878 score=+0.38 | ✅ 一致 |
| PU.1 | GM12878 p=1e-280 | GM12878 score=+0.28 | ✅ 一致 |
| IRF4 | GM12878 p=1e-200 | GM12878 score=+0.31 | ✅ 一致 |
| GATA1 | K562 p=1e-500 | K562 score=-0.45 | ✅ 一致 |
| TAL1 | K562 p=1e-380 | K562 score=-0.35 | ✅ 一致 |
| KLF1 | K562 p=1e-250 | K562 score=-0.22 | ✅ 一致 |
| CTCF | 两者均富集 | 差异 ≈ 0 (n.s.) | ✅ 组成型 |

**两种方法的结果高度一致，互相验证。** 这大大增强了我们对结论的信心。

但 footprint 分析提供了 motif 富集无法给出的额外信息：

1. **结合 vs 未结合的区分**：motif 存在不等于被结合，footprint 能区分
2. **同家族 TF 的区分**：GATA1 和 GATA2 的 motif 极其相似，但 TOBIAS 通过 footprint 深度和位点数量可以提供更细的区分
3. **结合位点的精确定位**：BINDetect 输出的 bound BED 文件给出了每个 TF 在每个条件中的精确结合位点坐标

## 实用技巧与注意事项

### 数据深度要求

💡 **Footprint 分析对数据深度有较高要求**
> 一般建议每个样本至少 **50M 有效 fragments**（去重后）才能获得可靠的 footprint 信号。如果你的数据量较少（<30M），footprint 的 V 形凹陷可能不够清晰，建议考虑使用下一篇介绍的 chromVAR 方法。

### 批量可视化

如果你想对一组感兴趣的 TF 批量生成 footprint 图：

```bash
# ============================================================
# 批量可视化 Top TF footprints
# ============================================================
for TF in EBF1 PAX5 PU.1 IRF4 GATA1 GATA2 TAL1 KLF1 CTCF; do
    TOBIAS PlotAggregate \
        --TFBS tobias/bindetect/${TF}/beds/${TF}_GM12878_bound.bed \
        --signals tobias/atacorrect/GM12878_merged_corrected.bw \
                  tobias/atacorrect/K562_merged_corrected.bw \
        --output figures/${TF}_footprint.pdf \
        --share_y both \
        --plot_boundaries \
        --title "${TF} Footprint"
done
```

### Footprint 分析的局限性

尽管 footprint 分析很强大，它也有局限：

1. **residence time 效应**：驻留时间短的 TF（如某些 pioneer factor 的快速扫描模式）可能不会留下清晰的 footprint
2. **小 motif 问题**：结合位点非常短的 TF（如 GATA 家族只有 ~6 bp）footprint 信号可能较弱
3. **计算资源需求**：完整的 TOBIAS 流程对大基因组数据集可能需要数小时
4. **需要多条件比较**：BINDetect 需要至少两个条件才能进行差异分析
