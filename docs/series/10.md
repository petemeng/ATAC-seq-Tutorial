## 为什么需要 Motif 富集分析

在第 9 篇中，你已经找到了 GM12878 和 K562 之间差异开放的染色质区域。但一个自然的问题是：**这些差异可及性的背后，是谁在驱动？**

转录因子（TF）与 DNA 的结合是序列特异性的——每个 TF 家族都倾向于识别一段特定的短序列模式，这就是 **motif**（序列基序）。如果我们发现某组差异 peak 中某个 TF 的 motif 显著富集，就可以推测：**这个 TF 可能正是导致这些区域开放的关键调控因子。**

💡 **类比理解**
> 想象你在一个城市里发现了很多被打开的锁。Motif 富集分析就是检查这些锁的锁孔形状——如果大部分锁孔都能被同一把钥匙打开，你就知道是谁干的了。

## Motif 基础知识

### PWM 与 PFM

一个 motif 并不是一段固定的序列，而是一个 **位置权重矩阵（PWM, Position Weight Matrix）**。它的前身是 **位置频率矩阵（PFM, Position Frequency Matrix）**，记录了每个位置上 A/C/G/T 出现的频率。

```
# PFM 示例：一个 8bp 的假想 motif
Pos:  1     2     3     4     5     6     7     8
A:  0.90  0.05  0.00  0.80  0.10  0.05  0.70  0.30
C:  0.03  0.85  0.00  0.05  0.10  0.05  0.10  0.30
G:  0.03  0.05  1.00  0.10  0.70  0.85  0.10  0.20
T:  0.04  0.05  0.00  0.05  0.10  0.05  0.10  0.20
```

PFM 经过背景校正和对数转换后就变成了 PWM，用于对任意序列进行打分。

### JASPAR 数据库

**JASPAR** 是目前最权威的开放获取 TF motif 数据库。2024 版本（JASPAR2024）包含了超过 2000 个经实验验证的脊椎动物 TF motif profile。我们将主要使用其 CORE vertebrates 集合。

### 已知 motif vs de novo motif

| 策略 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **已知 motif 富集** | 在数据库中搜索哪些已知 motif 在目标区域中过度表达 | 可直接关联到 TF，解释性强 | 只能找到数据库中有的 |
| **De novo motif 发现** | 从序列中从头搜索重复出现的模式 | 能发现新 motif | 计算量大，结果需人工解读 |

HOMER 和 MEME-ChIP 都同时支持这两种策略。

## 方法一：HOMER findMotifsGenome.pl

### 安装 HOMER

如果你还没有安装 HOMER：

```bash
# ============================================================
# 安装 HOMER
# ============================================================
conda install -c bioconda homer
# 安装 hg38 基因组数据
perl $(which configureHomer.pl) -install hg38
```

### 对 GM12878 特异性 peak 进行 motif 富集

```bash
# ============================================================
# HOMER motif 富集分析 —— GM12878 特异性 peak
# ============================================================
findMotifsGenome.pl \
    diffbind/results/GM12878_enriched_peaks.bed \
    hg38 \
    motif_results/GM12878/ \
    -size 200 \
    -mask \
    -p 8
```

参数解释：

| 参数 | 含义 |
|------|------|
| `diffbind/results/GM12878_enriched_peaks.bed` | 输入：GM12878 特异性差异开放 peak |
| `hg38` | 参考基因组 |
| `motif_results/GM12878/` | 输出目录 |
| `-size 200` | 以 peak 中心为原点，取 ±100 bp（共 200 bp）进行分析 |
| `-mask` | 屏蔽重复序列，避免假阳性 |
| `-p 8` | 使用 8 个线程 |

💡 **为什么用 `-size 200`**
> ATAC-seq 的 peak 宽度差异较大（几百到几千 bp）。使用固定 200 bp 窗口有两个好处：（1）聚焦于 peak 中心最可能的 TF 结合位点；（2）使所有 peak 的分析窗口大小一致，避免长 peak 对统计的过度影响。如果你想分析整个 peak 区域，可以用 `-size given`。

### 对 K562 特异性 peak 进行 motif 富集

```bash
# ============================================================

# ============================================================
```

## 方法二：MEME-ChIP

MEME Suite 是另一个经典的 motif 分析工具集。其中 `meme-chip` 整合了 MEME（de novo）、DREME（短 motif de novo）、CentriMo（中心富集分析）和 AME（已知 motif 富集）等多个子工具。

### 准备输入 FASTA

MEME-ChIP 需要 FASTA 格式的序列作为输入，而非 BED：

```bash
# ============================================================
# 提取差异 peak 序列 —— 为 MEME-ChIP 准备输入
# ============================================================
# GM12878 特异性 peak
bedtools getfasta \
    -fi ref/hg38.fa \
    -bed diffbind/results/GM12878_enriched_peaks.bed \
    -fo tmp/gm_peaks.fa

# K562 特异性 peak
bedtools getfasta \
    -fi ref/hg38.fa \
    -bed diffbind/results/K562_enriched_peaks.bed \
    -fo tmp/k562_peaks.fa
```

### 运行 MEME-ChIP

```bash
# ============================================================
# MEME-ChIP motif 分析 —— GM12878 特异性 peak
# ============================================================
meme-chip \
    -oc meme_results/GM12878 \
    tmp/gm_peaks.fa \
    -meme-nmotifs 10 \
    -db ref/JASPAR2024.meme

# ============================================================
# MEME-ChIP motif 分析 —— K562 特异性 peak
# ============================================================
meme-chip \
    -oc meme_results/K562 \
    tmp/k562_peaks.fa \
    -meme-nmotifs 10 \
    -db ref/JASPAR2024.meme
```

| 参数 | 含义 |
|------|------|
| `-oc` | 输出目录 |
| `-meme-nmotifs 10` | MEME de novo 搜索时最多返回 10 个 motif |
| `-db` | 已知 motif 数据库（JASPAR2024 MEME 格式） |

💡 **HOMER vs MEME-ChIP 怎么选**
> HOMER 速度更快、使用更简便，适合快速筛查。MEME-ChIP 提供了更丰富的子分析（特别是 CentriMo 中心富集分析，可以检验 motif 是否真的富集在 peak 中心附近）。建议：**先用 HOMER 快速跑一遍，再用 MEME-ChIP 做补充验证。**

## 背景序列的选择：一个容易被忽视的关键问题

Motif 富集分析的本质是一个统计检验：目标区域中某个 motif 的出现频率是否**显著高于**背景区域。因此，**背景怎么选，直接决定了你的结果是否可信。**

### 常见的背景选择策略

| 策略 | 做法 | 适用场景 | 注意事项 |
|------|------|----------|----------|
| **随机基因组区域** | HOMER 默认：随机抽取与目标 peak 数量、大小匹配的基因组区域 | 通用 | 可能包含大量不可及区域 |
| **所有 peak 作为背景** | 用全部 ATAC-seq peak 作为背景，检测差异 peak 中的额外富集 | 差异分析 | 更严格，推荐用于差异 peak 分析 |
| **GC 含量匹配** | 背景区域的 GC 含量分布与目标一致 | 启动子区域分析 | 消除 GC 偏差带来的假阳性 |

⚠️ **踩坑预警**
> 如果你用**全部基因组**作为背景来分析 ATAC-seq peak，几乎所有 TF motif 都会显著富集——因为开放染色质区域天然就比随机基因组区域包含更多 TF 结合位点。这时候你需要问自己：我要找的是"peak 比随机区域富集的 motif"还是"GM12878 peak 比 K562 peak 额外富集的 motif"？**后者才是差异分析中真正有意义的问题。**

### HOMER 中指定背景

```bash
# ============================================================
# 使用所有 peak 作为背景进行更严格的 motif 富集分析
# ============================================================
findMotifsGenome.pl \
    diffbind/results/GM12878_enriched_peaks.bed \
    hg38 \
    motif_results/GM12878_strict/ \
    -size 200 \
    -mask \
    -bg peaks/consensus/consensus_peaks.bed \
    -p 8
```

使用 `-bg` 参数指定背景 peak 集合后，富集结果会更加特异——只有真正在 GM12878 差异 peak 中额外富集的 motif 才会排在前面。

## 生物学解读

这是整个分析中最重要的一步——**把统计数字翻译成生物学故事。**

### GM12878 特异性 motif

**EBF1 和 PAX5 是 B 细胞谱系的核心转录因子**，它们在 GM12878 特异性开放区域中高度富集，完全符合预期。

- **EBF1**（Early B-cell Factor 1）：B 细胞发育的先驱因子（pioneer factor），能够直接打开封闭的染色质。它在 GM12878 差异 peak 中富集排名第一，说明这些区域的开放**可能是 EBF1 直接驱动的**。
- **PAX5**：B 细胞特异性的 paired-box 转录因子，维持 B 细胞身份。与 EBF1 协同调控 B 细胞基因表达程序。
- **PU.1/SPI1**：ETS 家族成员，在 B 细胞和髓系细胞中均有表达，但在 B 细胞中与 EBF1、PAX5 形成特异的调控网络。
- **IRF4**：干扰素调控因子家族成员，参与 B 细胞活化和浆细胞分化。

### K562 特异性 motif

**GATA1 是红系分化的主调控因子**，在 K562 特异性区域排名第一，说明差异可及性确实捕捉到了谱系特异的调控信息。

- **GATA1**：红系和巨核系发育的核心 TF，经典的先驱因子。K562 虽然是 CML 来源，但在体外培养条件下表现出明显的红系/巨核系特征。
- **TAL1/SCL**：bHLH 转录因子，常与 GATA1 形成复合体共同结合 DNA。HOMER 检测到的 `TAL1::GATA1` 复合 motif 正好反映了这一生物学事实。
- **KLF1**（EKLF）：红系特异的 Krüppel 样因子，调控 β-globin 等红系基因。
- **NFE2**：bZIP 家族，参与红系终末分化和血小板生成。

### 共有 motif

**CTCF** 在两组 peak 中均显著富集，这并不意外——CTCF 是一个普遍表达的绝缘子结合蛋白，参与三维基因组的拓扑域边界维持，与细胞谱系关系不大。

💡 **从 Motif 到 TF：注意逻辑**
> Motif 富集分析告诉你"**谁可能在这里结合**"，而不是"谁一定在这里结合"。同一个 motif 可能被同一 TF 家族的多个成员识别（比如 GATA1 和 GATA2 的 motif 非常相似）。要进一步区分是谁真正结合了，需要下一篇的 **TF footprinting** 分析，或者结合 ChIP-seq/CUT&Tag 数据。

## 实用技巧

### 可视化 Motif 富集结果

```r
# ============================================================
# R 可视化：读取 HOMER 结果并绘图
# ============================================================
library(ggplot2)

# 读取 HOMER 已知 motif 结果
gm_motifs <- read.delim("motif_results/GM12878/knownResults.txt")
k562_motifs <- read.delim("motif_results/K562/knownResults.txt")

# 取 Top 10，绘制柱状图
gm_top10 <- head(gm_motifs, 10)
gm_top10$logP <- -gm_top10$Log.P.value / log(10)
gm_top10$Motif <- factor(gm_top10$Motif.Name, levels = rev(gm_top10$Motif.Name))

ggplot(gm_top10, aes(x = logP, y = Motif)) +
    geom_bar(stat = "identity", fill = "#2166AC") +
    labs(title = "GM12878 特异性 Peak — Top 10 Motifs",
         x = "-log10(p-value)", y = "") +
    theme_minimal(base_size = 14)

ggsave("figures/GM12878_motif_enrichment.pdf", width = 8, height = 5)
```
